# UI Feature

## Check

- Reuse existing components and patterns.
- Use shadcn/ui when suitable.
- Do not introduce a new design system.
- Keep Tailwind consistent with existing tokens.
- Include loading, error, empty, success and disabled states.
- Check mobile and tablet layout.
- Check long client, service and master names.
- Use Russian text naturally.
- Format prices with `₽`.
- Format dates/times for `ru-RU` and `Europe/Moscow`.
- Check console and network errors through browser verification if available.
