# Next Feature Cycle

1. Choose one feature from `docs/feature-backlog.md`.
2. Create/switch to a branch.
3. Run the relevant prompt from `docs/codex-prompts.md`.
4. Wait for the plan.
5. Check scope.
6. Allow implementation.
7. Require tests.
8. Require browser verification.
9. Run pre-merge review.
10. Fix blocking issues.
11. Repeat.

## Rules

- One feature = one small reviewable diff.
- Do not take several big features at once.
- For appointments/RLS/security, always use specialized skills.
- For UI, always use `browser-qa` if available.
